# Card Game Skills

These skills adapt the general "skills as procedural memory" idea to the
`D:\Unity\card game` relay/autopilot stack.

Design rules:
- keep each skill narrow and tied to a repeated operational failure or workflow
- point to deterministic scripts instead of re-explaining long procedures
- make acceptance criteria inspectable through compact artifacts first
- avoid large-log reading unless a compact artifact is missing or proven wrong
